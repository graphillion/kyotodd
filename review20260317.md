# KyotoDD ソースコードレビュー (2026-03-17)

## 1. バグ・ロジックエラー

- [x] **[HIGH] `bddclosure_rec` のアルゴリズムが誤り** (`src/zdd_adv.cpp:601`)
  - `bddunion(C0, bddmeet(C0, C1))` は誤りで、正しくは `bddunion(C0, C1)` であるべき。
  - 例: F = {{a,b,v}} の下方閉包を計算すると {a,b},{a},{b},{∅} が欠落する。
  - `meet` (ペアワイズ集合積) ではなく `union` を使う必要がある。

- [x] **[HIGH] `ZDD_Import` 単一ルートのオーバーロードで未初期化 `bddp` を使用** (`src/bdd_io.cpp:301-304, 316-319`)
  - `bddimportz` の戻り値をチェックしていない。インポート失敗時に `p` が未初期化のまま `ZDD_ID(p)` に渡され、未定義動作となる。
  - 修正案: `bddp p = bddnull;` で初期化し、戻り値を確認する。

- [x] **[MEDIUM] `bddimply` に再帰深度制限がない** (`src/bdd_ops.cpp:397`)
  - `BDD_RecurGuard` も `bdd_gc_guard` も使用していない。深いBDDでスタックオーバーフローの危険がある。

- [x] **[MEDIUM] `bddimply` の戻り値 -1 が `&&` で真として扱われる** (`src/bdd_ops.cpp:445`)
  - `bddnull` が入力された場合、`bddimply` は -1 を返すが、`(-1) && bddimply(...)` は C++ で真と評価される。

- [x] **[LOW] `bddonset0` のドキュメントが「offset と同じ」と記載しているが実際は異なる** (`include/bdd_ops.h:248-253`)
  - `bddonset0` は変数 v を含む集合を選択し v を除去する（1-cofactor）。`bddoffset` は v を含まない集合を選択する。ドキュメントの記述が誤り。

- [x] **[LOW] 2-operand/3-operand キャッシュで `h == 0` の場合に false positive が発生し得る** (`src/bdd_base.cpp:93`)
  - 3-operand エントリの `h` が 0 の場合、2-operand のキャッシュ読み取りで誤ヒットする可能性がある。実際には稀だがアーキテクチャ上の潜在バグ。

## 2. メモリ安全性

- [x] **[HIGH] `bdd_gc_guard` が `overflow_error` 以外の例外で `bdd_gc_depth` をデクリメントしない** (`include/bdd_internal.h:32-51`)
  - `std::bad_alloc` 等が発生すると `bdd_gc_depth` がデクリメントされず、GC が永久に無効化される。
  - 修正案: RAII スコープガードを使用するか、`catch (...)` で `bdd_gc_depth` をデクリメントする。

- [x] **[MEDIUM] `bddgc_mark` / `bddsize_traverse` が無制限再帰** (`src/bdd_base.cpp:393-403, 459-466`)
  - `BDD_RecurGuard` なし。深い DAG でスタックオーバーフローの可能性。反復的実装への変更を推奨。

- [x] **[MEDIUM] `bddinit` 再初期化で部分的失敗した場合の use-after-free リスク** (`src/bdd_base.cpp:175-183`)
  - `std::free(bdd_cache)` の直後に `bdd_cache = nullptr; bdd_cache_size = 0;` を設定すべき。

- [x] **[MEDIUM] `bddsupport_collect` が無制限再帰** (`src/bdd_ops.cpp:451-459`)
  - 深い BDD でスタックオーバーフローの可能性がある。

- [x] **[MEDIUM] `bddcardmp16` にバッファオーバーフローのリスク** (`src/zdd_adv.cpp:803`)
  - `s != NULL` の場合にサイズチェックなしで `std::memcpy` を呼び出す。

- [x] **[MEDIUM] Python バインディング: `init()` を BDD/ZDD オブジェクト存在中に呼ぶと未定義動作** (`python/src/kyotodd/_binding.cpp:35-38`)
  - 既存オブジェクトが古いノード ID を保持したまま、ノードテーブルが解放される。

## 3. スレッド安全性

- [x] **[HIGH] 全グローバル状態がスレッド保護なし** (`include/bdd_types.h:48-77`, `include/bdd_internal.h:28`)
  - `bdd_nodes`, `bdd_node_used`, `bdd_varcount`, `bdd_unique_tables`, `bdd_cache`, `bdd_gc_depth`, `gc_roots()` 等がロックなし。
  - マルチスレッド使用時はデータ競合で未定義動作。シングルスレッド前提であればその旨を明記すべき。

- [ ] **[LOW] `ZDD_Random` が `std::rand()` を使用** (`src/zdd_ops.cpp:785`)
  - `std::rand()` はスレッドセーフでない。`<random>` の使用を推奨。

## 4. API 不整合

- [x] **[MEDIUM] `ZDD` にデフォルトコンストラクタがない** (`include/bdd_types.h:353`)
  - `BDD` は `BDD()` で `bddfalse` に初期化可能だが、`ZDD` は `explicit ZDD(int val)` のみ。`std::vector<ZDD> v(10);` がコンパイルできない。

- [x] **[MEDIUM] `BDD::operator<<` は `bddvar` を受けるが `ZDD::operator<<` は `int` を受ける** (`include/bdd_types.h:202, 444`)
  - `ZDD::Swap`, `ImplyChk`, `CoImplyChk` 等でも同様の不整合。負値が暗黙的に受け入れられる。

- [ ] **[LOW] `ZDD_LCM_*` が `char*` を引数に取る** (`include/bdd_ops.h:637-641`)
  - C++11 では文字列リテラルから `char*` への変換は非推奨。`const char*` に変更すべき。

- [ ] **[LOW] 未実装の ZDD メンバ関数が公開宣言されている** (`include/bdd_types.h:571-575`)
  - `PrintPla()`, `ZLev()`, `SetZSkip()` が `std::logic_error` を投げるだけ。削除するか `// not implemented` コメントを付けるべき。

- [x] **[LOW] `ZDD::ZLev` / `ZDD::SetZSkip` の未使用パラメータがコンパイル警告を出す** (`src/bdd_class.cpp:30-36`)
  - `(void)lev; (void)last;` パターンで抑制すべき。

## 5. エラーハンドリング

- [x] **[MEDIUM] エクスポート関数が書き込みエラーを無視** (`src/bdd_io.cpp:47-48`)
  - `fputs` や `std::ostream` のエラー状態をチェックしていない。切り詰められた出力を検出できない。

- [x] **[LOW] `bddvdump` の引数 `p` の null ポインタチェックがない** (`src/bdd_io.cpp:417`)

- [x] **[LOW] `bddgc_setthreshold` がバリデーションなし** (`src/bdd_base.cpp:385-387`)
  - 負値を含む任意の `double` を受け入れる。

- [ ] **[LOW] `bddvsize` のサイズ引数が `int` 型** (`include/bdd_base.h:114`)
  - 負値を許容する。`size_t` を使用すべき。

## 6. パフォーマンス

- [ ] **[HIGH] BDD/ZDD の構築・破棄のたびに `gc_roots()` の `unordered_set` を操作** (`include/bdd_types.h:149-170, 353-367`)
  - `(a & b) | (c ^ d)` のような式で複数の一時オブジェクトが GC ルートのハッシュセットにアクセスする。ホットパスに影響。

- [ ] **[MEDIUM] `node_index` が毎回バリデーションを実行** (`include/bdd_internal.h:55-65`)
  - 再帰演算のクリティカルパスで毎回2つの分岐チェック。リリースビルドでは `BDD_DEBUG_ASSERT` に移すことを推奨。

- [ ] **[MEDIUM] ムーブコンストラクタに `noexcept` がなく `std::vector` の再配置でコピーが使われる** (`include/bdd_types.h:164, 177, 361, 374`)
  - `bddgc_protect` が `unordered_set::insert` を呼ぶため `noexcept` にできない。GC ルート管理を例外を投げないデータ構造に変更すべき。

- [ ] **[LOW] `bddswap` が Shannon 分解を6回のコファクタ＋6回のAND＋3回のORで実装** (`src/bdd_ops.cpp:852-869`)
  - 専用の再帰 `bddswap_rec` にキャッシュを持たせれば大幅に高速化できる。

- [ ] **[LOW] `bddremainder` が結果をキャッシュしない** (`src/zdd_ops.cpp:773-781`)
  - 同一引数での繰り返し呼び出しが毎回再計算される。

- [ ] **[LOW] `bddlshift` / `bddrshift` が BDD を2回走査** (`src/bdd_ops.cpp:708, 762`)
  - バリデーション用に `bddsupport_collect` を呼び、その後シフト用に再走査。

- [ ] **[LOW] GC マーク配列に `uint8_t` を使用** (`src/bdd_base.cpp:410`)
  - ビットベクタにすればメモリ使用量が 1/8 になる。

## 7. C++11 準拠性

- [ ] **[MEDIUM] GoogleTest v1.14.0 は C++14 が必要** (`CMakeLists.txt:26`)
  - プロジェクトは C++11 を設定しているが、GoogleTest 1.14 は C++14 機能を使用する。GoogleTest 1.12.x に下げるべき。

- [ ] **[LOW] `#define BDD_RecurLimit 8192` はマクロ** (`include/bdd_types.h:44`)
  - `static constexpr int BDD_RecurLimit = 8192;` にすべき。

- [ ] **[LOW] `static const` ヘッダ変数は TU ごとにコピーが作成される** (`include/bdd_types.h:23-41, bdd_node.h:20-27`)
  - `constexpr` を使うとコンパイラの最適化が改善される。

- [ ] **[LOW] `CardMP16` が `malloc` で確保した `char*` を返す** (`include/bdd_types.h:561`)
  - C++ では `std::string` が適切。呼び出し元が `free` を呼ぶ必要がある。

- [ ] **[LOW] `CMAKE_CXX_EXTENSIONS OFF` が未設定** (`CMakeLists.txt`)
  - `-std=gnu++11` になる可能性があり、非標準拡張が見逃される。

## 8. テスト

- [ ] **[MEDIUM] `bddimport` の不正入力テストがない** (`test/test_bdd.cpp`)
  - 壊れたフォーマットや null ストリームに対するテストが不足。

- [ ] **[LOW] `capture_stdout` が POSIX 専用で戻り値チェックなし** (`test/test_bdd.cpp:6157-6176`)
  - `pipe()`, `dup()`, `dup2()` の戻り値を確認していない。Windows では動作しない。

- [ ] **[LOW] テストが内部グローバル変数 `bdd_gc_depth` を直接操作** (`test/test_bdd.cpp:5884-5890`)
  - 内部実装への依存。セマンティクス変更時にテストが無効化される可能性。

## 9. Python バインディング

- [ ] **[HIGH] `onset0` の docstring が誤り: 「offset と同じ」と記載** (`python/src/kyotodd/_binding.cpp:395-400`)
  - 実際は「変数 v を含む集合を選択し v を除去」する動作。offset（v を含まない集合の選択）とは異なる。

- [ ] **[HIGH] `onset` の docstring が誤り: 「v を除去」と記載** (`python/src/kyotodd/_binding.cpp:389-394`)
  - 実際は v を保持する。`onset0` が v を除去する方。

- [ ] **[LOW] `__mod__` の docstring に `%%` リテラル** (`python/src/kyotodd/_binding.cpp:351`)
  - `help()` 出力で `%%` がそのまま表示される。`%` に修正すべき。

- [ ] **[LOW] `len` プロパティ名が Python ユーザーに誤解を招く** (`python/src/kyotodd/_binding.cpp:620-621`)
  - Python では `len` は要素数を連想するが、実際は「ファミリ内の最大集合サイズ」。`max_set_size` 等が望ましい。

- [ ] **[LOW] `ZDD.__invert__` が未定義** (`python/src/kyotodd/_binding.cpp`)
  - `BDD` には `~x` 構文があるが `ZDD` にはない。

- [ ] **[LOW] `BDD(42)` のような無効値に対するバリデーションがない** (`python/src/kyotodd/_binding.cpp:123-129`)
