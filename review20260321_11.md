[x] **[MEDIUM] `BDD.support()` の説明が「変数の論理積」になっており、実装・ヘッダ・既存テストが示す「変数の論理和」と食い違っています。**
該当: `docs/api/python/bdd.rst:183-187`, `docs/tutorials/bdd_basics.rst:82-93`, `python/src/kyotodd/_core.pyi:160-161`, `include/bdd_types.h:325-326`, `src/bdd_ops.cpp:490-494`, `test/test_bdd.cpp:1210-1212`。
Python API ページとチュートリアルは `support()` を「conjunction of variables」と説明していますが、公開ヘッダと型スタブはどちらも「disjunction of variables」と書いており、実装も `bddfalse` から `hi=bddtrue` の鎖を積むことで `x1 | x2 | ...` を作っています。実テストでも `support(x1 & x2) == (x1 | x2)` を期待しています。集合としての support を BDD でどう表すかは API 契約そのものなので、現行ドキュメントの「論理積」という説明は利用者を誤誘導します。
→ 対応: `bdd.rst` と `bdd_basics.rst` の "conjunction" → "disjunction" に修正。
